## 6. React 구현 어려운 부분 체크리스트

### 6.1 기술적 난제 분류

#### 🔴 High Risk (React 구현 매우 어려움)

| 항목 | 설명 | 대안 | 확인 방법 |
|------|------|------|----------|
| **ActiveX 컨트롤** | OCX, DLL 기반 기능 | iframe 유지 또는 대체 라이브러리 | .ocx 파일 검색 |
| **Trustform 특수 기능** | XForms 확장 기능 | 재설계 필요 | xmlns:trf 검색 |
| **WebSocket 미지원** | 실시간 양방향 통신 | Socket.io 또는 HTTP polling | EJB 검색 |
| **Direct DB Access** | 클라이언트측 SQL | API 래핑 필요 | SQL 직접 호출 검색 |

#### 🟠 Medium Risk (주의 필요)

| 항목 | 설명 | 대안 | 확인 방법 |
|------|------|------|----------|
| **복잡한 마스터-디테일** | 3단계 이상 중첩 | 상태관리 라이브러리 | 화면 구조 분석 |
| **실시간 데이터 갱신** | polling 기반 | SWR/React Query | refetch 로직 검색 |
| **복잡한 폼 검증** | cross-field validation | React Hook Form + Zod | validation 로직 검색 |
| **대용량 그리드** | 10,000+ 행 처리 | 가상화 (react-window) | 그리드 라이브러리 분석 |

#### 🟢 Low Risk (표준 구현 가능)

| 항목 | React 대안 | 구현 난이도 |
|------|-----------|------------|
| 일반 CRUD | React + REST API | 쉬움 |
| 팝업/모달 | React Portal | 쉬움 |
| 탭 인터페이스 | react-tabs | 쉬움 |
| 기본 검증 | HTML5 validation | 쉬움 |

### 6.2 화면별 체크리스트 (분석 시 필수 확인)

```
[화면 분석 체크리스트]

□ 프론트엔드
  □ ActiveX/OCX 사용 여부
  □ WebSocket 사용 여부
  □ XForms 특수 속성 사용 여부
  □ 마스터-디테일 중첩 단계
  □ 실시간 데이터 갱신 빈도
  □ 복잡한 폼 검증 존재 여부
  □ 대용량 데이터 처리 여부

□ 백엔드
  □ EJB Session Bean 사용 여부
  □ 직접 SQL 호출 여부
  □ 비즈니스 로직 위치 (프론트/백)
  □ API 현대화 필요 여부
  □ 인증/권한 방식
```
