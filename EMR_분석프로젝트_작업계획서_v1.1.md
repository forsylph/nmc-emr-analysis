## 5. 백엔드 전환율 평가 기준 (구체화)

### 5.1 평가 기준

```
[백엔드 전환율 산정 공식]

백엔드 전환율 = Σ(화면별 전환 가중치) ÷ 총 분석 화면 수 × 100

- 임계값: 10%
- 측정 시점: Phase 3 완료 후 (M4)
```

### 5.2 화면별 전환 가중치 산정

| 항목 | 판정 기준 | 가중치 | 측정 방법 |
|------|----------|--------|----------|
| **EJB 구조 변경** | Session Bean → REST API 전환 필요 | 1.0 | ejb-jar.xml 분석 |
| **데이터 모델 변경** | 테이블 구조 변경 필요 | 1.0 | Entity 클래스 분석 |
| **비즈니스 로직 이동** | 프론트 → 백엔드 로직 이전 | 0.8 | JS→Java 이전 분석 |
| **API 신규 개발** | 기존 API 미지원 기능 | 0.5 | API 매핑 테이블 |
| **인증/권한 변경** | 보안 정책 변경 | 0.7 | security 설정 분석 |

**화면별 전환율 계산 예시**:
```
SMPMO00300:
- EJB 구조 변경: 1.0
- 비즈니스 로직 이동: 0.8
- API 신규 개발: 0.5 (2개)
= (1.0 + 0.8 + 1.0) ÷ 5 = 0.56 (56% 가중치)
```

## 6. 분석→설계 인계 프로세스

### 6.1 인계 Flow

```
┌────────────┐    ┌────────────┐    ┌────────────┐    ┌────────────┐
│   분석     │ → │   리뷰     │ → │   설계     │ → │   인계     │
│  (개발팀)  │    │  (피어리뷰) │    │  (피어리뷰)│    │  (산출물)  │
└────────────┘    └────────────┘    └────────────┘    └────────────┘
      │                 │                 │                 │
      ▼                 ▼                 ▼                 ▼
  화면 분석서       품질 검토        컴포넌트 설계      최종 산출물
  기능 매핑표       체크리스트       기술 스펙          아카이브
```

### 6.2 단계별 산출물

| 단계 | 산출물 | 책임자 | 품질 기준 |
|------|--------|--------|----------|
| **분석** | 화면 분석서, 기능 매핑표 | 개발자 | 체크리스트 100% |
| **리뷰** | 리뷰 의견서, 수정 요청 | 피어리뷰팀 | Pass/수정요청/재검토 |
